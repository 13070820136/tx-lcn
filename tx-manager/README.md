# LCN分布式事务框架的设计原理 

框架基于spring事务框架的基础之上做的再次封装，通过控制协调本地事务与全局事务的一致性从而达到分布式事务的效果。

特别说明：LCN框架 非TCC机制，非2PC机制 非3PC机制。


## LCN名称的得来

在设计框架之初的1.0 ~ 2.0的版本时，框架设计的步骤是如下的，各取其首字母得来的LCN命名。

锁定事务单元（lock）   
确认事务模块状态(confirm)   
通知事务(notify)
